# V0.13 — analyse IA optionnelle des traces xAPI

## Objectif

La V0.13 ajoute une analyse IA optionnelle des traces xAPI lues depuis TRAX/LRS.

L'objectif est d'aider un formateur ou un pilote de cours à interpréter plus rapidement les indicateurs déjà disponibles dans `Suivi xAPI` :

- synthèse pédagogique ;
- ressources à surveiller ;
- ressources activées sans trace ;
- tests ou activités avec échecs fréquents ;
- recommandations d'amélioration du cours ;
- limites de l'analyse.

L'IA reste une aide à l'interprétation. Elle ne remplace pas le formateur et ne doit pas prendre de décision automatique sur les apprenants.

## Branche de travail

```text
v0.13-ai-xapi-analysis
```

## Configuration Mistral / Vibe

Pour une clé Mistral / Vibe, la configuration de test recommandée dans le plugin est :

```text
Fournisseur IA : mistral
URL API IA    : https://api.mistral.ai/v1/chat/completions
Modèle IA     : mistral-small-latest
Timeout IA    : 20
Anonymisation : strict
```

Alternative possible pour un modèle plus puissant, selon les droits du compte et le quota disponible :

```text
Modèle IA : mistral-large-latest
```

La clé API ne doit jamais être saisie dans Git ni dans la documentation. Elle doit être fournie côté serveur via :

```text
ITXEB_AI_API_KEY
```

## Test attendu

Quand l'analyse IA est activée, que l'URL API IA, le modèle et la variable serveur sont configurés, le bouton `Tester configuration IA` envoie uniquement un prompt technique minimal. Il ne transmet aucune trace xAPI réelle.
