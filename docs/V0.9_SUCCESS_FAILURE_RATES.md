# V0.9 — Taux réussite / échec

## Objectif

Ce lot améliore la lecture pédagogique des tests dans l’onglet `Analyse`.

Avant ce lot, la colonne `Tests` affichait uniquement :

```text
x réussis / y échoués
```

Après ce lot, elle affiche aussi les taux calculés :

```text
x réussis / y échoués — Réussite n % / Échec n %
```

## Vue concernée

La modification concerne l’onglet :

```text
Analyse
```

Elle s’applique aux ressources qui disposent de tentatives de test.

## Calcul

Pour chaque ressource :

```text
taux réussite = test_passed / test_attempts * 100
taux échec    = test_failed / test_attempts * 100
```

Le calcul protège contre la division par zéro.

Si aucune tentative n’est présente, la colonne garde la valeur :

```text
-
```

## Source des données

La source reste l’outbox locale `evnt_evhk_itxeb_out` via le repository analytics.

Aucune migration SQL n’est nécessaire.

## Interaction avec les filtres

Les taux respectent les filtres existants :

- période ;
- ressource ;
- type d’objet.

Si `Type = Test (tst)` est sélectionné, la table Analyse affiche uniquement les ressources de type test, avec les taux correspondants.

## Implémentation

La modification est injectée pendant l’installation du companion UIHook par le script :

```text
scripts/patch_course_ui_success_rates.php
```

Le script d’installation appelle ce patcher après le patch du filtre type :

```text
scripts/install_course_ui_companion.sh
```

Le patch est idempotent : si les libellés `Réussite` et `Échec` sont déjà présents dans le fichier généré, il ne réapplique rien.
