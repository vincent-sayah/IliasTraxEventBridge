# V0.13 — consolidation du compagnon UI

## Objectif

La V0.13 vise à consolider techniquement le plugin compagnon `IliasTraxEventBridgeCourseUI` après la stabilisation fonctionnelle de la V0.12.

La V0.12 est publiée sur `main` avec le tag stable `v0.12.0`. La V0.13 repart de cette base stable.

## Branche de travail

```text
v0.13-consolidation-ui-companion
```

## Version de développement

```text
0.13.0-dev
```

## Problème à traiter

La V0.12 fonctionne, mais plusieurs évolutions du compagnon UI sont appliquées par patchers successifs lors de l'installation :

- navigation Delos ;
- route Info ;
- lecture directe TRAX/LRS ;
- vues LRS primaires ;
- configuration outbox technique ;
- dashboard pédagogique ;
- apprenants en difficulté ;
- export CSV Expert enrichi ;
- layout V0.12.

Ce fonctionnement est acceptable pour la V0.12, mais il rend la maintenance plus complexe à moyen terme.

## Objectif technique V0.13

Réintégrer proprement les modifications stables dans les templates du plugin compagnon :

```text
companion/IliasTraxEventBridgeCourseUI/plugin.php.tpl
companion/IliasTraxEventBridgeCourseUI/classes/*.php.tpl
```

Le script d'installation doit continuer à installer le compagnon, mais il doit dépendre de moins en moins de patchers successifs.

## Principe d'architecture conservé

```text
Outbox locale = file technique d'envoi
TRAX/LRS      = source officielle du suivi xAPI pédagogique
```

La consolidation V0.13 ne doit pas modifier ce principe.

## Périmètre V0.13

### Inclus

- intégration directe dans les templates des modifications V0.12 stabilisées ;
- réduction du nombre de patchers nécessaires après installation ;
- simplification du script `install_course_ui_companion_with_standalone_fix.sh` ;
- conservation stricte du rendu fonctionnel V0.12 ;
- documentation de la nouvelle procédure d'installation ;
- validation de non-régression sur VM ILIAS.

### Hors périmètre

- nouvelle analyse IA ;
- nouveaux indicateurs pédagogiques ;
- modification du modèle de données ;
- modification du principe TRAX/LRS comme source pédagogique ;
- suppression des diagnostics V0.11 ;
- refonte graphique majeure.

## Script d'audit V0.13

Un script d'audit est ajouté pour objectiver l'état du compagnon installé avant consolidation :

```text
scripts/audit_course_ui_companion_v013.sh
```

Il permet de vérifier :

- la présence du template source ;
- la présence du fichier compagnon actif installé ;
- la version et la branche du plugin ;
- les marqueurs fonctionnels V0.12 présents dans le fichier actif ;
- les marqueurs déjà présents dans le template source ;
- les patchers encore référencés par le wrapper d'installation ;
- la syntaxe PHP du compagnon ;
- un extrait de diff entre template et fichier actif.

### Exécution sur VM ILIAS

```bash
cd /var/www/ilias/public/Customizing/global/plugins/Services/EventHandling/EventHook/IliasTraxEventBridge

git fetch origin
git checkout v0.13-consolidation-ui-companion
git pull origin v0.13-consolidation-ui-companion

bash scripts/audit_course_ui_companion_v013.sh
```

### Sortie attendue

Le script doit montrer quels marqueurs sont présents dans le fichier actif mais absents du template. Ces écarts indiquent les blocs à réintégrer proprement dans les templates `.tpl`.

## Critères d'acceptation

La V0.13 sera validée si :

- le plugin indique `0.13.0` en version finale ;
- le compagnon s'installe depuis les templates consolidés ;
- les vues `Tableau de bord`, `Analyse`, `Expert` et `Configuration` restent fonctionnellement identiques à la V0.12 ;
- la synthèse pédagogique reste visible dans `Tableau de bord` et `Analyse` ;
- les statuts `OK`, `À surveiller`, `Critique`, `Désactivée` restent cohérents ;
- les couleurs orange et rouge restent visibles ;
- l'export PDF reste fonctionnel ;
- l'export CSV Expert conserve les colonnes pédagogiques V0.12 ;
- le bloc `Apprenants en difficulté` reste présent ;
- les diagnostics V0.11 restent accessibles ;
- le script d'installation reste idempotent ;
- les contrôles `php -l` sont OK.

## Plan de travail proposé

1. Identifier les patchers encore nécessaires après installation.
2. Comparer le fichier compagnon actif généré avec les templates source.
3. Réinjecter les blocs stabilisés directement dans les templates `.tpl`.
4. Simplifier progressivement le wrapper d'installation.
5. Conserver temporairement les patchers uniquement pour compatibilité ou migration.
6. Tester une installation fraîche du compagnon sur VM ILIAS.
7. Créer une checklist `VALIDATION_0.13.md`.
8. Promouvoir vers `main` après validation.

## Point de vigilance

La V0.13 ne doit pas chercher à ajouter de nouvelles fonctionnalités visibles. Elle doit rendre la base V0.12 plus propre, plus maintenable et plus simple à installer.
