# V0.9 — Filtre par type d’objet

## Objectif

Ce lot ajoute le filtre `Type` aux vues de feedback cours.

Il complète les filtres déjà présents :

- période ;
- ressource ;
- type d’objet ILIAS.

## Vues concernées

Le filtre est disponible dans :

- `Tableau de bord` ;
- `Analyse` ;
- `Expert` ;
- export CSV Expert.

## Différence entre ressource et type

Le filtre `Ressource` cible un objet ILIAS précis.

Exemple :

```text
Ressource = test — tst — ref_id 216
```

Dans ce cas, les indicateurs portent uniquement sur cette ressource.

Le filtre `Type` cible une famille d’objets ILIAS.

Exemple :

```text
Type = Test (tst)
```

Dans ce cas, les indicateurs portent sur tous les tests du cours.

## Règle ergonomique

Le filtre `Type` est utile uniquement quand `Ressource = Toutes les ressources`.

Si une ressource précise est sélectionnée :

- le filtre `Type` est désactivé visuellement ;
- le filtre `Type` est ignoré techniquement ;
- l’export CSV Expert n’applique pas de filtre type supplémentaire.

Cette règle évite les combinaisons contradictoires, par exemple :

```text
Ressource = blog ref_id 221
Type = Test (tst)
```

## Fonctionnement

Le filtre est appliqué côté interface avant l’appel au repository d’analytics.

Le principe est identique au filtre par ressource : la liste des ressources du cours est réduite avant le calcul des indicateurs.

Aucune migration SQL n’est nécessaire.

## Valeur par défaut

Par défaut, le filtre vaut :

```text
Tous les types
```

Dans ce cas, aucune restriction par type n’est appliquée.

## Types affichés

La liste est construite depuis les ressources traçables réellement présentes dans le cours.

Exemples de types :

- `blog` ;
- `file` ;
- `frm` ;
- `htlm` ;
- `lm` ;
- `mcst` ;
- `sahs` ;
- `tst` ;
- `webr` ;
- `wiki`.

Les libellés sont affichés sous forme lisible, par exemple :

```text
Test (tst)
Blog (blog)
Fichier (file)
Wiki (wiki)
```

## Export CSV Expert

Quand `Ressource = Toutes les ressources` et qu’un type est sélectionné, l’export CSV Expert reprend le filtre type.

Le fichier exporté inclut :

- le `filter_ref_id` si une ressource est sélectionnée ;
- le `filter_obj_type` si un type est sélectionné sans ressource précise.

Le nom du fichier contient aussi le type filtré, par exemple :

```text
itxeb_course_210_type_tst_expert_YYYYMMDD_HHMMSS.csv
```

Si une ressource précise est sélectionnée, le type est ignoré et le fichier reste centré sur la ressource :

```text
itxeb_course_210_ref_216_expert_YYYYMMDD_HHMMSS.csv
```

## Implémentation

Le filtre est injecté pendant l’installation du companion UIHook par le script :

```text
scripts/patch_course_ui_type_filter.php
```

Le script d’installation appelle ce patcher après la copie des templates :

```text
scripts/install_course_ui_companion.sh
```

Le patch est idempotent : si le filtre type d’objet est déjà présent dans le fichier généré, il ne le réapplique pas.
