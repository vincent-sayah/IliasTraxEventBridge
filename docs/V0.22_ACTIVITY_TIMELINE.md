# V0.22 — Activité dans le temps

## Objectif

Le bloc actuel `Activité par jour` peut devenir trop long lorsque la période sélectionnée couvre plusieurs semaines ou plusieurs mois.

La V0.22 remplace cette liste par un bloc plus compact :

```text
Activité dans le temps
```

## Problème constaté

Sur une période longue, une ligne par jour occupe une grande partie de la page du tableau de bord.

Cela gêne la lecture des autres blocs :

- synthèse pédagogique ;
- questions à fort taux d'échec ;
- actions ;
- top ressources ;
- ressources sans activité.

## Solution proposée

Ajouter un sélecteur d'affichage propre au bloc activité :

```text
7 jours | 14 jours | 30 jours | Par semaine | Détail complet
```

## Comportement prévu

| Cas | Affichage |
|---|---|
| Période courte | affichage quotidien compact, par défaut 14 jours |
| Période longue | regroupement automatique par semaine |
| Détail complet | détail quotidien dans un bloc repliable `<details>` |

## Résumé affiché

Le bloc affiche des indicateurs synthétiques :

- activité affichée ;
- nombre de périodes actives ;
- périodes sans activité ;
- pic d'activité ;
- moyenne par période.

## Avantages

- Le tableau de bord reste court.
- L'utilisateur peut choisir la granularité de lecture.
- Les longues périodes ne saturent plus la page.
- Le détail complet reste disponible.
- La solution ne nécessite pas de nouvelle table SQL.
- La solution ne nécessite pas de JavaScript complexe.

