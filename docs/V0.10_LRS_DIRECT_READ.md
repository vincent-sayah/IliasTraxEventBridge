# V0.10 — Lecture directe TRAX / LRS

## Objectif

Ajouter une lecture directe du LRS TRAX depuis l'écran `Suivi xAPI` du cours.

La V0.10 fournit une vue directe du contenu TRAX pour le cours courant : nombre de statements retournés, pagination, verbes et ressources.

## Décision d'architecture

La comparaison local / TRAX est retirée pour le moment.

La table locale `evnt_evhk_itxeb_out` est une outbox technique. Elle peut être vidée en exploitation. TRAX conserve alors les statements déjà reçus, mais la base locale de comparaison disparaît. Une comparaison entre outbox locale et TRAX peut donc devenir trompeuse.

Le bloc `TRAX / LRS direct` n'affiche donc plus d'écart local / TRAX ni de statut de cohérence.

## Principe retenu

- l'envoi continue d'utiliser l'outbox locale existante ;
- la lecture utilise un client read-only dédié ;
- aucune ligne SQL locale n'est modifiée pendant la lecture ;
- la vue LRS reste exploitable même si l'outbox locale est vide.

## Client ajouté

```text
classes/class.ilIliasTraxEventBridgeLrsReadClient.php
```

Ce client effectue uniquement :

```text
GET /statements
```

avec :

```text
Accept: application/json
X-Experience-API-Version: 1.0.3
```

Le client sait suivre une URL `more` renvoyée par le LRS. Si l'URL `more` est relative, elle est reconstruite avec l'hôte de l'endpoint `/statements` configuré.

## Synthèse cours

```text
classes/class.ilIliasTraxEventBridgeLrsCourseSummary.php
```

La synthèse interroge le LRS avec l'activité xAPI du cours :

```text
<ILIAS_BASE_URL>/xapi/activity/course/ref/<course_ref_id>
```

Paramètres utilisés :

```text
activity=<activité cours>
related_activities=true
since=<date début période>
limit=100
```

`related_activities=true` permet de retrouver les statements où le cours est présent dans `contextActivities.parent`.

## Affichage dashboard

Le bloc affiche :

```text
TRAX / LRS direct
```

avec :

- état LRS ;
- code HTTP ;
- nombre de statements TRAX retournés ;
- nombre de pages LRS lues ;
- statut de pagination ;
- période `since` utilisée ;
- éventuel `more` restant ;
- activité cours xAPI ;
- table des verbes retournés par TRAX ;
- table des ressources retournées par TRAX.

## Pagination LRS

La synthèse suit automatiquement le champ `more` retourné par TRAX.

Limite actuelle :

```text
5 pages maximum
100 statements par page
```

## Détail par ressource côté LRS

La synthèse agrège les statements retournés par TRAX par ressource.

La ressource est identifiée en priorité par :

```text
/xapi/extensions/ref_id
```

Sinon, elle est regroupée par `statement.object.id`.

Le tableau affiche :

```text
Ressources retournées par TRAX
```

Colonnes :

- titre ;
- object id xAPI ;
- type ILIAS ;
- ref_id ;
- nombre de statements.

## Fonctions retirées

Les fonctions suivantes sont retirées pour le moment :

- comparaison local / TRAX globale ;
- comparaison local / TRAX par ressource ;
- divergence par statement UUID.

Une future comparaison fiable nécessitera une table locale durable des statements envoyés, distincte de l'outbox technique.
