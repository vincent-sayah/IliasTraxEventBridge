# V0.10 / V0.10.1 — Suivi xAPI alimenté par TRAX / LRS

## 1. Statut

| Élément | Valeur |
|---|---|
| Version stable actuelle | `0.10.1` |
| Branche | `v0.10-lrs-direct-read` |
| Version initiale de la fonctionnalité | `0.10.0` |
| Version corrective stable | `0.10.1` |
| Source officielle du suivi xAPI | TRAX/LRS |
| Rôle de l'outbox locale | File d'envoi technique |

La V0.10.1 reprend l'architecture V0.10 et ajoute la correction stable du fichier `sql/dbupdate.php` avec le marqueur ILIAS `<#1>`.

## 2. Décision d'architecture

À partir de la V0.10, le suivi xAPI du cours utilise TRAX/LRS comme source fonctionnelle.

```text
Outbox locale = file d'envoi technique
TRAX/LRS      = source officielle du suivi xAPI
```

L'outbox `evnt_evhk_itxeb_out` sert uniquement à envoyer les statements et à superviser l'envoi. Elle ne doit plus alimenter les vues pédagogiques du cours, car elle peut être purgée en exploitation.

## 3. Vues concernées

Les vues suivantes sont alimentées par TRAX/LRS :

```text
Tableau de bord
Analyse
Expert
Export CSV Expert
Export PDF Tableau de bord
Comparaison entre périodes
```

Les données locales ne sont plus utilisées pour analyser l'activité pédagogique du cours.

## 4. Organisation des onglets

```text
Tableau de bord
→ KPI pédagogiques, activité par jour, actions xAPI, top ressources, ressources sans statement TRAX, export PDF.

Analyse
→ analyse des ressources, apprenants en difficulté, verbes retournés par TRAX, ressources retournées par TRAX.

Expert
→ statements TRAX détaillés, source TRAX, statement ID, export CSV.

Configuration
→ activation xAPI, personnalisation du tableau de bord, supervision technique de l'envoi, diagnostic LRS simple.
```

## 5. Export PDF

L'onglet `Tableau de bord` contient un bouton :

```text
Export PDF
```

Le rapport PDF est construit à partir du modèle TRAX/LRS et ne dépend pas de l'outbox locale.

Contenu du rapport :

```text
Cours
Période
Filtre ressource
Filtre type
Synthèse KPI
Activité par jour
Actions xAPI
Ressources
```

La génération PDF utilise l'ordre suivant :

```text
1. Dompdf si disponible côté Composer
2. wkhtmltopdf si disponible côté serveur
3. rapport HTML imprimable si aucun moteur PDF n'est disponible
```

Chemin `wkhtmltopdf` supporté notamment :

```text
/opt/wkhtmltopdf/bin/wkhtmltopdf
```

## 6. Supervision technique de l'envoi

La supervision de l'outbox locale est déplacée dans l'onglet `Configuration`.

Section :

```text
Supervision technique de l'envoi xAPI
```

Elle affiche uniquement l'état de la file locale d'envoi :

```text
generated
sending
sent
failed
other
```

Cette section ne sert pas au suivi pédagogique. Elle permet seulement de vérifier que l'envoi vers TRAX fonctionne.

## 7. Diagnostic LRS

Le bloc `TRAX / LRS direct` est affiché dans l'onglet `Configuration`.

Il conserve le diagnostic de lecture :

```text
État LRS
Statements TRAX
Pages LRS
Pagination LRS
Période depuis
More LRS restant
Activité cours xAPI
```

Les détails fonctionnels `Verbes retournés par TRAX` et `Ressources retournées par TRAX` sont affichés dans l'onglet `Analyse`.

## 8. Éléments retirés du tableau de bord pédagogique

Les blocs suivants sont retirés du tableau de bord :

```text
État technique local
TRAX / LRS direct
```

Ces blocs relèvent du diagnostic technique, pas du tableau de bord pédagogique xAPI.

## 9. Client LRS

Classe :

```text
classes/class.ilIliasTraxEventBridgeLrsReadClient.php
```

Ce client effectue uniquement :

```text
GET /statements
```

Avec les en-têtes xAPI :

```text
Accept: application/json
X-Experience-API-Version: 1.0.3
```

Il utilise l'authentification Basic HTTP déjà configurée pour TRAX.

## 10. Modèle analytique LRS

Classe :

```text
classes/class.ilIliasTraxEventBridgeLrsCourseSummary.php
```

Cette classe construit le modèle de données utilisé par les vues du cours :

- synthèse globale ;
- activité par jour ;
- comparaison entre périodes ;
- répartition par verbe ;
- regroupement par ressource ;
- lignes Expert ;
- pagination LRS.

La requête LRS utilise l'activité xAPI du cours :

```text
<ILIAS_BASE_URL>/xapi/activity/course/ref/<course_ref_id>
```

Paramètres :

```text
activity=<activité cours>
related_activities=true
since=<date début période>
limit=100
```

`related_activities=true` permet de retrouver les statements où le cours est présent dans `contextActivities.parent`.

## 11. Comparaison entre périodes

La comparaison entre périodes est calculée avec TRAX/LRS.

Elle lit une fenêtre double :

```text
période sélectionnée x 2
```

Puis sépare :

```text
période actuelle
période précédente de même durée
```

Elle ne dépend plus de l'outbox locale.

## 12. Pagination

Le client suit le champ `more` retourné par TRAX.

Limite actuelle :

```text
5 pages maximum
100 statements par page
```

Si cette limite est atteinte, la vue doit indiquer que la pagination LRS n'est pas complète.

## 13. Ressources

Les statements sont regroupés par ressource.

La ressource est identifiée en priorité par l'extension xAPI :

```text
/xapi/extensions/ref_id
```

Sinon, le regroupement utilise :

```text
statement.object.id
```

## 14. Patchs UI

Le companion applique les patchs suivants :

```text
scripts/patch_course_ui_lrs_direct_summary.php
scripts/patch_course_ui_lrs_primary_views.php
scripts/patch_course_ui_outbox_technical_config.php
scripts/patch_course_ui_lrs_diagnostics_config.php
scripts/patch_course_ui_lrs_analysis_details.php
scripts/patch_course_ui_pdf_export.php
scripts/patch_course_ui_pdf_route.php
scripts/patch_course_ui_pdf_wkhtmltopdf_paths.php
```

Ces patchs ajoutent la lecture LRS, font de TRAX/LRS la source primaire des vues pédagogiques, déplacent la supervision technique dans `Configuration`, déplacent le diagnostic LRS simple dans `Configuration`, déplacent les détails `Verbes retournés par TRAX` / `Ressources retournées par TRAX` dans `Analyse`, et ajoutent l'export PDF du tableau de bord.

## 15. Fonctions non retenues

Les fonctions suivantes ne sont pas retenues à ce stade :

- comparaison local / TRAX globale ;
- comparaison local / TRAX par ressource ;
- divergence par statement UUID.

Elles ne seront réintroduites que si une table locale durable d'archive des statements envoyés est ajoutée, distincte de l'outbox technique purgeable.
