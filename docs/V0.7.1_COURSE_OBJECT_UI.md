# V0.7.1 — Configuration xAPI depuis l'objet cours

## Objectif

Déplacer l'usage quotidien de la sélection des ressources xAPI depuis l'administration globale du plugin vers l'objet cours lui-même.

L'administrateur ou responsable d'un cours doit pouvoir configurer les traces xAPI directement depuis le cours, sans passer par :

```text
Administration > Plugins > IliasTraxEventBridge > Configurer
```

L'administration globale du plugin peut conserver un accès de secours, mais elle ne doit plus être le chemin principal pour sélectionner les ressources.

## Besoin fonctionnel V0.7.1

Depuis un objet cours ILIAS, un administrateur de cours doit disposer d'une entrée visible, par exemple :

```text
TRAX / xAPI
```

ou :

```text
Configuration xAPI
```

Cette entrée doit permettre :

- d'activer ou désactiver les traces xAPI du cours ;
- de cocher les ressources traçables ;
- d'enregistrer les choix dans `evnt_evhk_itxeb_ccfg` et `evnt_evhk_itxeb_rcfg` ;
- de réutiliser le filtrage V0.7 déjà validé ;
- de limiter l'accès aux utilisateurs ayant les droits d'administration ou modification du cours.

## Architecture retenue

Le plugin actuel est un plugin EventHook :

```text
Services/EventHandling/EventHook/IliasTraxEventBridge
```

Il reste le plugin principal, responsable de :

- la réception des événements ILIAS ;
- le journal debug ;
- l'outbox ;
- le cron ;
- l'envoi TRAX ;
- le filtrage avant outbox ;
- les tables `ccfg` et `rcfg`.

Pour afficher une entrée dans l'objet cours, la V0.7.1 introduit un plugin compagnon d'interface :

```text
Services/UIComponent/UserInterfaceHook/IliasTraxEventBridgeCourseUI
```

Ce plugin compagnon reste léger. Il ne duplique pas la logique métier. Il sert uniquement à exposer l'écran V0.7 dans le contexte du cours.

- réutiliser les classes du plugin principal.

Le lot 4 injecte une entrée visible, sous forme de bouton flottant :

```text

```

Conditions d'affichage :

```text

plugin principal disponible

```

Le bouton est injecté par `ilIliasTraxEventBridgeCourseUIUIHookGUI::getHTML()` et pointe vers l'URL préparée par le bridge :


Le lot 4 ne rend pas encore l'écran complet. Le clic peut donc seulement recharger la page avec ces paramètres. Le rendu de l'écran complet est prévu au lot 5.

## Règles d'accès

L'accès à la configuration doit être réservé aux utilisateurs qui ont au moins un des droits suivants sur le cours :

```text
write
edit_permission
manage_members
```

Ces droits sont déjà utilisés par `ilIliasTraxEventBridgeCourseTrackingGUI` et doivent être conservés.

## Parcours utilisateur cible

1. L'administrateur entre dans un cours ILIAS.
2. Il voit une entrée `TRAX / xAPI` ou `Configuration xAPI`.
3. Il ouvre l'écran de configuration.
4. L'écran affiche le cours courant sans demander de saisir manuellement le `course_ref_id`.
5. Il coche `Activer les traces xAPI pour ce cours`.
6. Il sélectionne les ressources à tracer.
7. Il enregistre.
8. Les tables `ccfg` et `rcfg` sont mises à jour.
9. Le filtrage V0.7 existant applique immédiatement ces choix.

## Critères de validation

### Accès UI

- L'entrée xAPI est visible dans un cours pour un administrateur de cours.
- L'entrée xAPI n'est pas visible, ou refuse l'accès, pour un utilisateur sans droits suffisants.
- L'écran s'ouvre sans devoir saisir `course_ref_id` manuellement.

### Écriture configuration

Après enregistrement depuis l'objet cours :

```sql
SELECT *
FROM evnt_evhk_itxeb_ccfg
WHERE course_ref_id = 194;

SELECT course_ref_id, ref_id, obj_id, obj_type, enabled, updated_at, updated_by
FROM evnt_evhk_itxeb_rcfg
WHERE course_ref_id = 194
ORDER BY ref_id;
```

Résultat attendu :

```text
cours activé si coché
ressources cochées enabled = 1
ressources non cochées enabled = 0
```

### Filtrage outbox conservé

Ressource activée :

```text
outbox créée puis envoyée
```

Ressource désactivée :

```text
aucune ligne outbox
```

### Non-régression V0.7.0

- Le cron continue de fonctionner.
- Les statements `file_downloaded` et `repository_object_access` sont conservés.
- Les enrichissements V0.6 sont conservés.
- Aucun `root` ou `crs` parasite n'est généré.
