# V0.9 — Apprenants en difficulté anonymisés

## Objectif

Ce lot ajoute dans l’onglet `Analyse` un bloc permettant de repérer les apprenants qui semblent en difficulté sur les tests.

La vue est volontairement anonymisée : elle ne doit pas afficher de nom, de login ou d’adresse courriel.

## Vue concernée

La modification concerne l’onglet :

```text
Analyse
```

Un bloc est ajouté sous le tableau des ressources :

```text
Apprenants en difficulté
```

## Données affichées

Le bloc affiche au maximum 10 apprenants.

Colonnes :

- apprenant anonymisé ;
- nombre d’alertes ;
- nombre d’échecs ;
- nombre de scores faibles ;
- score moyen ;
- dernière alerte ;
- ressources concernées.

## Anonymisation

L’identifiant affiché est un pseudonyme calculé à partir du `user_id` ILIAS.

Exemple :

```text
Apprenant a1b2c3d4
```

Le `user_id` réel n’est pas affiché dans cette vue.

## Règles de détection MVP

Une trace de test produit une alerte apprenant si :

```text
success = false
ou verb_id contient failed
ou score_raw < 50
```

Un apprenant est affiché dans le bloc si :

```text
échecs >= 2
ou scores faibles >= 2
ou alertes >= 3
```

## Source des données

La source reste l’outbox locale `evnt_evhk_itxeb_out`, via les `expert_rows` déjà calculés par le repository analytics.

Aucune migration SQL n’est nécessaire.

## Interaction avec les filtres

La vue respecte les filtres existants :

- période ;
- ressource ;
- type d’objet.

Exemples :

```text
Type = Test (tst)
```

Le bloc est calculé uniquement sur les tests visibles dans l’analyse.

```text
Ressource = test ref_id 216
```

Le bloc est calculé uniquement sur cette ressource.

## Implémentation

La modification est injectée pendant l’installation du companion UIHook par le script :

```text
scripts/patch_course_ui_struggling_learners.php
```

Le script d’installation appelle désormais les patchers dans cet ordre :

```text
1. patch_course_ui_type_filter.php
2. patch_course_ui_success_rates.php
3. patch_course_ui_failure_signals.php
4. patch_course_ui_struggling_learners.php
```

Le contrôle `php -l` reste exécuté après les patchs.

## Limites MVP

La vue exploite uniquement les 200 dernières traces détaillées disponibles dans `expert_rows`.

Elle ne remplace pas une analyse complète TRAX/LRS et ne doit pas être utilisée comme décision pédagogique automatique. Elle sert de signal d’attention pour le formateur ou l’administrateur du cours.
