# V0.7 — Résolution des ressources traçables d'un cours

## Objectif du lot 3

Ce lot prépare les données nécessaires à la future interface cours `TRAX / xAPI`.

Il ajoute un resolver PHP qui peut, à partir d'un `course_ref_id` :

- vérifier que la référence est bien un cours ILIAS ;
- parcourir l'arbre du cours ;
- détecter les ressources traçables ;
- récupérer `ref_id`, `obj_id`, `obj_type`, titre et chemin affichable ;
- joindre l'état de configuration existant depuis `evnt_evhk_itxeb_rcfg` ;
- retourner une structure exploitable par la future UI.

Ce lot ne modifie pas encore :

- l'interface cours ;
- l'écran d'administration global ;
- le filtrage avant outbox ;
- la génération des statements xAPI.

## Classe ajoutée

```text
classes/class.ilIliasTraxEventBridgeCourseResourceResolver.php
```

Méthodes publiques principales :

```php
resolveCourse(int $courseRefId): array
findTrackableResources(int $courseRefId): array
getTrackableTypes(): array
```

## Types de ressources détectés

Le resolver prépare les types déjà utiles pour V0.6/V0.7 :

```text
file  -> file
tst   -> test
blog  -> blog
wiki  -> wiki
webr  -> web_link
mcst  -> media
frm   -> forum
htlm  -> html_module
lm    -> learning_module
sahs  -> scorm
```

Les dossiers, catégories et autres conteneurs ne sont pas considérés comme traçables, mais le resolver descend dans l'arbre pour trouver les ressources traçables imbriquées.

## Structure retournée pour un cours

`resolveCourse($courseRefId)` retourne une structure de ce type :

```php
[
    'course_ref_id' => 177,
    'course_obj_id' => 594,
    'course_title' => 'test V0.7',
    'course_configured' => true,
    'course_enabled' => true,
    'course_config' => [...],
    'resources' => [...]
]
```

## Structure retournée pour une ressource

Chaque ressource retournée contient :

```php
[
    'course_ref_id' => 177,
    'ref_id' => 182,
    'obj_id' => 605,
    'obj_type' => 'wiki',
    'resource_family' => 'wiki',
    'title' => 'test',
    'path' => 'Dossier / test',
    'configured' => true,
    'enabled' => true,
    'config' => [...]
]
```

## Jointure avec la configuration V0.7

Le resolver lit la configuration existante via :

```text
ilIliasTraxEventBridgeCourseTrackingRepository
```

Il joint les lignes de :

```text
evnt_evhk_itxeb_rcfg
```

par `course_ref_id` et `ref_id`.

Si aucune configuration de ressource n'existe encore, les champs sont :

```text
configured = false
enabled = false
config = []
```

Cela permettra à l'interface V0.7 d'afficher clairement les ressources non encore configurées.

## Stratégie de parcours

Le resolver utilise d'abord l'arbre ILIAS :

```php
$GLOBALS['DIC']->repositoryTree()->getChilds($refId)
```

Puis, par sécurité, il dispose d'un fallback SQL sur la table `tree` si l'objet arbre n'est pas disponible.

Le parcours est récursif avec une limite de profondeur pour éviter les boucles.

## Test fonctionnel provisoire

Ce lot ne fournit pas encore d'écran cours. Le resolver sera consommé dans le lot suivant.

Pour une vérification manuelle, on peut créer un script temporaire depuis la racine ILIAS une fois l'autoload reconstruit. Le test devra appeler :

```php
$resolver = new ilIliasTraxEventBridgeCourseResourceResolver();
$result = $resolver->resolveCourse(177);
print_r($result);
```

Le résultat attendu est une liste de ressources du cours avec les types comme `file`, `tst`, `blog`, `wiki`, `webr`, `mcst`, `htlm`.

## Critères de réussite du lot 3

- La nouvelle classe est présente.
- La syntaxe PHP est valide.
- Les types traçables sont centralisés.
- Le resolver ne crée pas de ligne outbox.
- Le resolver ne modifie pas les tables de configuration.
- Les futures données UI sont disponibles : `ref_id`, `obj_id`, `obj_type`, titre, chemin, état `configured/enabled`.
