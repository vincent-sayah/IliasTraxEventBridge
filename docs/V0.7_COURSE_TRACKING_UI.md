# V0.7 — Interface cours TRAX / xAPI

## Objectif

Ce jalon ajoute une première interface de configuration TRAX / xAPI pour un cours ILIAS.

L'objectif est de permettre à un responsable de cours de préparer les choix suivants :

- activer ou désactiver les traces xAPI pour le cours ;
- activer ou désactiver les traces xAPI ressource par ressource ;
- enregistrer ces décisions dans les tables V0.7 ;
- respecter les permissions de modification du cours.

## Fichier ajouté

```text
classes/class.ilIliasTraxEventBridgeCourseTrackingGUI.php
```

## Classe ajoutée

```php
ilIliasTraxEventBridgeCourseTrackingGUI
```

Annotation Ctrl préparée :

```php
@ilCtrl_IsCalledBy ilIliasTraxEventBridgeCourseTrackingGUI: ilObjCourseGUI
```

Cette classe est donc prévue pour être appelée depuis le contexte d'un objet cours.

## Commandes UI disponibles

```text
show
save
enableAll
disableAll
resetCourse
```

### show

Affiche :

- le résumé du cours ;
- l'état de configuration du cours ;
- le nombre de ressources traçables ;
- le nombre de ressources configurées ;
- le nombre de ressources activées ;
- le tableau des ressources retournées par `ilIliasTraxEventBridgeCourseResourceResolver`.

### save

Enregistre :

- `course_enabled` dans `evnt_evhk_itxeb_ccfg` ;
- les ressources cochées dans `evnt_evhk_itxeb_rcfg` ;
- les ressources non cochées comme désactivées dans `evnt_evhk_itxeb_rcfg`.

### enableAll

Active le cours et toutes les ressources traçables détectées.

### disableAll

Désactive le cours et toutes les ressources traçables détectées.

### resetCourse

Supprime toute la configuration du cours :

- ligne cours dans `evnt_evhk_itxeb_ccfg` ;
- lignes ressources associées dans `evnt_evhk_itxeb_rcfg`.

## Tables utilisées

```text
evnt_evhk_itxeb_ccfg
evnt_evhk_itxeb_rcfg
```

## Permissions

La modification est autorisée si l'utilisateur dispose d'au moins un de ces droits sur le cours :

```text
write
edit_permission
manage_members
```

Sans ce droit, l'écran passe en lecture seule et les actions d'écriture refusent la modification.

## Limite volontaire de ce jalon

Ce jalon ajoute l'écran contrôleur et l'écriture de configuration, mais ne filtre pas encore la génération xAPI.

Le filtrage sera ajouté dans le jalon suivant.

Donc après ce lot :

```text
- les choix cours/ressources sont enregistrables ;
- les tables V0.7 sont alimentées ;
- les statements continuent encore à être générés comme en V0.6 jusqu'au lot de filtrage.
```

## Point d'intégration ILIAS à valider

Le plugin est un plugin `EventHook`. La classe GUI est prête pour un appel depuis `ilObjCourseGUI`, mais l'accrochage visuel exact dans l'écran du cours dépend des points d'extension disponibles dans ILIAS 10.

Deux possibilités restent à valider sur serveur :

```text
Objet cours > Paramètres > section TRAX / xAPI
```

ou :

```text
Objet cours > onglet / sous-écran TRAX / xAPI
```

Si ILIAS 10 ne permet pas une injection propre depuis un plugin EventHook seul, il faudra ajouter un mécanisme complémentaire ou un point d'entrée compatible.

## Validation serveur

Depuis le répertoire du plugin :

```bash
find . -name "*.php" -print0 | xargs -0 -n1 php -l
```

Puis :

```bash
cd /var/www/ilias
sudo -u apache composer du
sudo -u apache php cli/setup.php build --yes
```

Contrôle présence de la classe :

```bash
grep -R "CourseTrackingGUI" classes docs CHANGELOG.md -n
grep -R "enableAll\|disableAll\|resetCourse\|course_enabled" classes/class.ilIliasTraxEventBridgeCourseTrackingGUI.php -n
```

Contrôle SQL après enregistrement d'un cours :

```sql
SELECT *
FROM evnt_evhk_itxeb_ccfg
ORDER BY updated_at DESC
LIMIT 10;

SELECT course_ref_id, ref_id, obj_id, obj_type, enabled, updated_at, updated_by
FROM evnt_evhk_itxeb_rcfg
ORDER BY updated_at DESC, course_ref_id, ref_id
LIMIT 30;
```
