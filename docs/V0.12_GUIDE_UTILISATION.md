# Guide d’utilisation V0.12 — dashboard pédagogique

## Objectif

La V0.12 transforme le suivi xAPI du cours en outil d’aide au pilotage pédagogique.

La V0.11 servait principalement à fiabiliser l’exploitation, le diagnostic et la connexion TRAX/LRS. La V0.12 ajoute une lecture pédagogique des traces afin d’aider un formateur, un concepteur ou un pilote de cours à repérer rapidement les ressources utilisées, les ressources peu ou pas utilisées, les échecs et les signaux faibles.

## Principe d’architecture

La règle d’architecture reste inchangée :

```text
Outbox locale = file technique d’envoi
TRAX/LRS      = source officielle du suivi xAPI pédagogique
```

Les indicateurs pédagogiques affichés dans le suivi de cours sont calculés à partir des données TRAX/LRS.

## Accès

Depuis un cours ILIAS :

```text
Cours > Suivi xAPI
```

Les onglets principaux sont :

```text
Configuration
Tableau de bord
Analyse
Expert
```

## Tableau de bord

L’onglet `Tableau de bord` donne une vue synthétique du cours sur la période sélectionnée.

Il affiche notamment :

- le nombre de statements TRAX/LRS analysés ;
- le nombre d’apprenants actifs ;
- le nombre de ressources utilisées ;
- le nombre de ressources sans statement TRAX ;
- le nombre de pages LRS lues ;
- le score moyen si des scores sont disponibles ;
- le nombre de ressources critiques ;
- le nombre de ressources à surveiller.

## Synthèse pédagogique

La V0.12 ajoute un bloc `Synthèse pédagogique` visible dans le tableau de bord et dans l’onglet Analyse.

Ce bloc affiche quatre compteurs principaux :

| Indicateur | Signification |
|---|---|
| OK | Ressources utilisées sans signal pédagogique défavorable. |
| À surveiller | Ressources avec un signal faible, par exemple absence de trace ou score moyen à surveiller. |
| Critiques | Ressources avec un signal fort, par exemple taux d’échec élevé ou score moyen faible. |
| Sans trace | Ressources activées mais sans statement TRAX/LRS sur la période. |

La synthèse ajoute aussi des phrases simples, par exemple :

```text
41 trace(s) TRAX/LRS analysée(s) sur la période.
12 apprenant(s) actif(s) détecté(s).
1 ressource(s) sont en statut critique.
3 ressource(s) sont à surveiller.
```

Cette synthèse est déterministe : elle ne dépend pas d’une IA et ne fait pas appel à un service externe.

## Statuts pédagogiques

Chaque ressource peut recevoir un statut pédagogique.

| Statut technique | Libellé affiché | Usage |
|---|---|---|
| `ok` | OK | Ressource utilisée sans signal défavorable. |
| `watch` | À surveiller | Ressource présentant un signal faible ou incomplet. |
| `critical` | Critique | Ressource présentant un signal fort d’échec ou de difficulté. |
| `disabled` | Désactivée | Ressource désactivée dans la configuration xAPI du cours. |

## Règles de calcul V0.12

Les règles de calcul sont volontairement simples et lisibles.

| Condition | Statut obtenu |
|---|---|
| Ressource désactivée | `disabled` |
| Ressource activée sans trace TRAX/LRS | `watch` |
| Taux d’échec supérieur ou égal à 50 % avec au moins 2 échecs | `critical` |
| Taux d’échec supérieur ou égal à 30 % avec au moins 1 échec | `watch` |
| Score moyen inférieur à 50 % | `critical` |
| Score moyen inférieur à 70 % | `watch` |
| Aucun signal défavorable | `ok` |

Ces seuils sont des seuils V0.12. Ils pourront être ajustés dans une version ultérieure.

## Onglet Analyse

L’onglet `Analyse` détaille les ressources du cours avec les informations pédagogiques calculées.

Les colonnes principales sont :

| Colonne | Description |
|---|---|
| Statut | Badge OK, À surveiller, Critique ou Désactivée. |
| Raison | Explication courte du statut. |
| Ressource | Titre ou chemin de la ressource. |
| Type | Type d’objet ILIAS. |
| xAPI | Indique si la ressource est activée ou désactivée. |
| Traces | Nombre de statements associés. |
| Apprenants | Nombre d’apprenants distincts. |
| Dernière trace | Dernier événement observé. |
| Score moyen | Score moyen calculé si disponible. |
| Tests | Nombre de réussites et d’échecs. |
| Taux échec | Taux d’échec calculé pour la ressource. |

Les ressources critiques ou à surveiller sont remontées visuellement afin de faciliter la lecture par le formateur.

## Apprenants en difficulté

La V0.12 conserve le bloc `Apprenants en difficulté` dans l’onglet Analyse.

Ce bloc est anonymisé : aucun nom ni courriel n’est affiché. Les apprenants sont présentés sous forme de pseudonymes techniques.

Le bloc permet d’identifier les apprenants qui cumulent des signaux négatifs, par exemple :

- plusieurs échecs ;
- plusieurs scores faibles ;
- plusieurs alertes sur des tests.

Ce bloc sert à attirer l’attention du formateur. Il ne doit pas être utilisé seul pour prendre une décision pédagogique sans vérification complémentaire.

## Onglet Expert

L’onglet `Expert` reste une vue détaillée des traces.

Il sert principalement à :

- contrôler les statements reçus ;
- vérifier les verbes xAPI ;
- vérifier les ressources concernées ;
- exporter les données pour analyse complémentaire ;
- diagnostiquer les écarts entre ILIAS et TRAX/LRS.

## Export CSV Expert enrichi

La V0.12 enrichit l’export CSV Expert avec des colonnes pédagogiques.

Les colonnes historiques sont conservées :

```text
date
course_ref_id
filter_ref_id
user_id
verb_label
verb_id
resource_title
ref_id
obj_id
obj_type
score_raw
completion
success
status
outbox_id
statement_uuid
last_error
```

Les nouvelles colonnes V0.12 sont :

```text
pedagogical_status
pedagogical_label
pedagogical_reason
resource_failure_rate
resource_avg_score_raw
resource_traces
resource_learners_count
resource_is_critical
resource_is_watch
```

### Exemple d’interprétation

Une ressource de test peut apparaître comme :

```text
pedagogical_status      = critical
pedagogical_label       = Critique
pedagogical_reason      = Taux d’échec élevé sur cette ressource.
resource_failure_rate   = 82.4
resource_avg_score_raw  = 10.34
resource_traces         = 29
resource_learners_count = 12
resource_is_critical    = oui
resource_is_watch       = non
```

Cela signifie que les traces TRAX/LRS indiquent un taux d’échec élevé sur cette ressource pour la période sélectionnée.

## Filtres

Les vues `Tableau de bord`, `Analyse` et `Expert` utilisent les filtres disponibles dans l’interface :

- période ;
- ressource ;
- type d’objet lorsque le filtre est présent.

Les indicateurs et exports doivent toujours être interprétés par rapport à la période et aux filtres sélectionnés.

## Points de vigilance

Les indicateurs V0.12 ne remplacent pas l’analyse pédagogique humaine.

Un statut `Critique` signifie qu’une ressource mérite une attention prioritaire, mais il ne dit pas à lui seul pourquoi les apprenants échouent. Les causes peuvent être multiples : difficulté de contenu, consigne ambiguë, problème technique, mauvaise configuration, ou manque de prérequis.

Un statut `À surveiller` signifie qu’un signal faible existe, mais qu’il doit être vérifié avant action.

## Validation rapide

Après déploiement V0.12, vérifier :

```text
Cours > Suivi xAPI > Tableau de bord
```

Résultat attendu :

```text
Synthèse pédagogique visible
Compteurs OK / À surveiller / Critiques / Sans trace visibles
```

Puis vérifier :

```text
Cours > Suivi xAPI > Analyse
```

Résultat attendu :

```text
Tableau des ressources avec Statut et Raison
Bloc Apprenants en difficulté visible
```

Enfin vérifier :

```text
Cours > Suivi xAPI > Expert > Export CSV
```

Résultat attendu :

```text
Colonnes pedagogical_status, pedagogical_label, pedagogical_reason et resource_failure_rate présentes dans le CSV.
```
