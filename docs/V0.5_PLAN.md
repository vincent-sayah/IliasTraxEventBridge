# Plan V0.5 — périmètre cours et activation par cours

## Objectif principal

La V0.5 doit garantir que les traces xAPI envoyées vers TRAX concernent uniquement les objets contenus dans un objet **cours**.

Un objet placé directement dans une catégorie, par exemple un test créé hors cours, ne doit pas générer de statement xAPI envoyé vers TRAX.

## Règles métier

1. Le plugin peut continuer à journaliser les événements bruts.
2. La création d'un statement dans l'outbox doit être bloquée si l'objet concerné n'appartient pas à un cours.
3. Le cours parent doit être identifié à partir du `ref_id`, de l'arborescence ILIAS ou du contexte disponible dans l'événement.
4. Si aucun cours parent n'est trouvé, l'événement reste non éligible xAPI.

## Paramétrage au niveau du cours

Ajouter une configuration accessible à l'administrateur du cours :

- activer ou désactiver l'envoi xAPI vers TRAX pour le cours ;
- choisir les types d'objets activés pour la génération de traces ;
- appliquer cette configuration avant la création du statement outbox.

## Types d'objets à couvrir

Déjà couverts en V0.4 :

- test ;
- fichier.

À ajouter en V0.5 :

- blog ;
- forum ;
- lien web ;
- mediacast ;
- wiki ;
- module web ;
- module SCORM.

## Critères de validation

- Un test placé dans une catégorie ne génère pas de statement envoyé vers TRAX.
- Le même test placé dans un cours peut générer un statement si le cours l'autorise.
- Un cours avec xAPI désactivé ne génère aucun statement pour ses objets.
- Un cours avec xAPI activé uniquement pour les fichiers ne génère pas de statement pour les autres types d'objets.
- Les nouveaux types d'objets listés sont détectés et filtrés par la configuration du cours.
