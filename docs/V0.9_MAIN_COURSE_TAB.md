# V0.9 — Onglet principal Suivi xAPI

## Objectif

Le lien `Suivi xAPI` ne doit plus apparaître comme sous-onglet de `Paramètres`.

Il doit apparaître comme onglet principal du cours, au même niveau que les onglets standards du cours, par exemple :

```text
Contenu | Membres | Suivi xAPI | Paramètres
```

## Vue concernée

La modification concerne le companion UIHook :

```text
IliasTraxEventBridgeCourseUI
```

## Comportement attendu

Quand l’utilisateur dispose des droits de gestion du cours :

- l’onglet `Suivi xAPI` est affiché dans la barre principale du cours ;
- l’onglet est inséré après `Membres` quand cet onglet est détecté ;
- si `Membres` n’est pas détecté, l’onglet est inséré avant `Paramètres` ;
- si aucun emplacement connu n’est détecté, un lien de secours est ajouté dans le contenu de la page.

## Comportement supprimé

Le companion ne doit plus ajouter `Suivi xAPI` dans les sous-onglets de `Paramètres`.

La méthode `modifyGUI()` ne modifie donc plus les `sub_tabs`.

## Implémentation

L’injection est réalisée dans :

```text
companion/IliasTraxEventBridgeCourseUI/classes/class.ilIliasTraxEventBridgeCourseUIUIHookGUI.php.tpl
```

Méthode principale :

```text
injectCourseMainTabIntoHtml()
```

Cette méthode recherche d’abord un onglet `Membres`, puis un onglet `Paramètres`, puis applique un lien de secours si nécessaire.

## Activation visuelle

Quand l’utilisateur est dans une vue `Suivi xAPI`, l’onglet reçoit une classe `active` afin d’être distingué comme onglet courant quand le thème ILIAS l’interprète.
