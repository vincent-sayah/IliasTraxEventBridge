# V0.9 — Feedback cours et tableau de bord xAPI

## Objectif

La V0.9 ajoute une fonctionnalité de feedback directement dans la fenêtre `Suivi xAPI` d’un cours ILIAS.

L’objectif n’est pas de remplacer TRAX ou Kibana, mais de donner à l’administrateur du cours une lecture simple des traces xAPI générées par son cours : activité, ressources utilisées, erreurs d’envoi et traces détaillées.

## Principe retenu

La V0.9 démarre avec trois vues préformatées plutôt qu’un constructeur de rapport libre :

| Vue | Public | Objectif |
|---|---|---|
| Synthèse / Tableau de bord | administrateur de cours | comprendre rapidement si le cours génère de l’activité |
| Analyse | formateur / responsable pédagogique | identifier les ressources utilisées, peu utilisées ou problématiques |
| Expert | support / administrateur technique | contrôler les traces locales et l’état outbox |

## Source des données

Le premier jalon V0.9 lit les données depuis l’outbox locale `evnt_evhk_itxeb_out`.

Avantages :

- pas de nouvel appel TRAX côté affichage ;
- pas de dépendance réseau supplémentaire ;
- cohérence avec ce que le plugin a réellement généré ;
- diagnostic possible même si l’envoi LRS est en erreur.

Limite assumée : cette vue affiche les traces générées localement par le plugin, pas une requête complète sur l’historique du LRS.

## Indicateurs V0.9 MVP

### Tableau de bord

- nombre total de traces xAPI ;
- nombre de traces envoyées ;
- nombre de traces en erreur ;
- nombre d’apprenants actifs, comptés sans afficher leur identité ;
- nombre de ressources utilisées ;
- score moyen quand les tests fournissent un score ;
- activité par jour ;
- distribution par verbe xAPI ;
- top ressources ;
- état technique local par statut outbox.

### Analyse

Table par ressource :

- signal pédagogique ou technique ;
- titre / chemin ;
- type ILIAS ;
- état xAPI activé ou désactivé ;
- nombre de traces ;
- nombre d’apprenants ;
- dernière trace ;
- temps moyen si disponible ;
- score moyen si disponible ;
- résultat des tests si disponible.

### Expert

Table des 200 dernières traces locales du cours :

- date ;
- user_id ILIAS ;
- verbe xAPI ;
- ressource ;
- type ;
- score ;
- completion ;
- success ;
- statut outbox ;
- id outbox ;
- statement UUID ;
- dernière erreur.

### Export CSV expert

L’onglet `Expert` expose un bouton `Exporter CSV`.

L’export reprend la période sélectionnée et produit un fichier UTF-8 avec BOM, séparé par `;`, afin de faciliter l’ouverture sous Excel ou LibreOffice depuis Windows.

Colonnes exportées :

- date ;
- course_ref_id ;
- user_id ;
- verb_label ;
- verb_id ;
- resource_title ;
- ref_id ;
- obj_id ;
- obj_type ;
- score_raw ;
- completion ;
- success ;
- status ;
- outbox_id ;
- statement_uuid ;
- last_error.

## Périodes prévues

Les vues d’analyse proposent une période simple :

- 7 jours ;
- 30 jours ;
- 90 jours ;
- 365 jours.

La période par défaut est 30 jours.

## Confidentialité

La vue Synthèse ne doit pas afficher les noms des apprenants.

La vue Expert affiche uniquement l’identifiant utilisateur ILIAS (`user_id`) pour permettre le diagnostic technique.

L’export CSV Expert suit la même règle : il exporte le `user_id` ILIAS, mais pas le nom ou l’adresse courriel de l’apprenant.
