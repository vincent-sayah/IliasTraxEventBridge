# V0.7 — Accès visible à la configuration xAPI par cours

## Objectif

Ce jalon rend l'écran de configuration xAPI par cours accessible dans ILIAS depuis l'administration du plugin.

L'objectif fonctionnel final reste :

```text
Objet cours > Paramètres ou onglet TRAX / xAPI
```

Mais, comme le plugin actuel est un `EventHook`, l'injection directe dans les paramètres natifs d'un cours dépend des points d'extension réellement disponibles dans ILIAS 10.

Ce jalon fournit donc un accès visible, stable et testable depuis :

```text
Administration > Plugins > IliasTraxEventBridge > Configurer
```

## Accès utilisateur

Dans l'écran d'administration du plugin, une nouvelle section est affichée :

```text
Configuration xAPI par cours
```

Elle permet de saisir un :

```text
course_ref_id
```

puis d'ouvrir l'écran :

```text
TRAX / xAPI — configuration du cours
```

## Actions disponibles

L'écran permet :

```text
- d'activer ou désactiver les traces xAPI pour le cours ;
- d'activer ou désactiver les ressources traçables du cours ;
- de tout activer ;
- de tout désactiver ;
- de réinitialiser la configuration du cours.
```

## Données écrites

L'activation globale du cours est écrite dans :

```text
evnt_evhk_itxeb_ccfg
```

L'activation par ressource est écrite dans :

```text
evnt_evhk_itxeb_rcfg
```

## Commandes GUI ajoutées

Dans `ilIliasTraxEventBridgeConfigGUI`, les commandes suivantes délèguent vers `ilIliasTraxEventBridgeCourseTrackingGUI` :

```text
configureCourseTracking
saveCourseTracking
enableAllCourseTracking
disableAllCourseTracking
resetCourseTracking
```

`ilIliasTraxEventBridgeCourseTrackingGUI` peut désormais être embarqué par un autre contrôleur grâce à un `linkTarget` et une table de correspondance des commandes.

## Limite volontaire

Ce jalon ne filtre toujours pas les statements xAPI.

Tant que le jalon de filtrage n'est pas livré :

```text
- les choix sont enregistrés ;
- les tables de configuration sont alimentées ;
- la génération xAPI conserve le comportement V0.6.
```

Le filtrage avant insertion dans `evnt_evhk_itxeb_out` sera traité dans le jalon suivant.

```bash

git fetch origin --prune --tags
git pull --ff-only origin v0.7

git status --short
git log --oneline --decorate -10
grep -n '\$version' plugin.php

find . -name "*.php" -print0 | xargs -0 -n1 php -l
```

Puis reconstruire ILIAS :

```bash
cd /var/www/ilias
sudo -u apache composer du
sudo -u apache php cli/setup.php build --yes
```

## Validation UI

Dans ILIAS :

```text
Administration > Extensibilité > Plugins > IliasTraxEventBridge > Configurer
```

ou selon le libellé exact ILIAS :

```text
Administration > Plugins > IliasTraxEventBridge > Configurer
```

Vérifier la présence de la section :

```text
Configuration xAPI par cours
```

Saisir le `ref_id` d'un cours contenant des ressources traçables, puis cliquer sur :

```text
Ouvrir la configuration xAPI du cours
```

L'écran attendu est :

```text
TRAX / xAPI — configuration du cours
```

Il doit afficher :

```text
- le titre du cours ;
- course_ref_id ;
- course_obj_id ;
- l'état xAPI du cours ;
- la liste des ressources traçables ;
- les boutons d'enregistrement et d'actions rapides.
```

## Validation SQL après enregistrement

Après avoir coché le cours et quelques ressources puis enregistré :

```sql
SELECT *
FROM evnt_evhk_itxeb_ccfg
ORDER BY updated_at DESC
LIMIT 10;

SELECT course_ref_id, ref_id, obj_id, obj_type, enabled, updated_at, updated_by
FROM evnt_evhk_itxeb_rcfg
ORDER BY updated_at DESC, course_ref_id, ref_id
LIMIT 30;
```

Attendu :

```text
- une ligne dans evnt_evhk_itxeb_ccfg pour le cours ;
- plusieurs lignes dans evnt_evhk_itxeb_rcfg pour les ressources ;
- enabled = 1 pour les ressources cochées ;
- enabled = 0 pour les ressources non cochées ;
- updated_by renseigné.
```
