- `git status --short` vide sur serveur et Windows.
- Tables V0.7 présentes :
  - `evnt_evhk_itxeb_ccfg` ;
  - `evnt_evhk_itxeb_rcfg`.
- Accès admin plugin opérationnel :

```sql
SELECT *
FROM evnt_evhk_itxeb_ccfg
WHERE course_ref_id = 194;
```

Configuration ressources :

```sql
SELECT course_ref_id, ref_id, obj_id, obj_type, enabled, updated_at, updated_by
FROM evnt_evhk_itxeb_rcfg
WHERE course_ref_id = 194
ORDER BY ref_id;
```

Outbox récente :

```sql
SELECT id, event_log_id, event_type, obj_type, ref_id, obj_id,
       user_id, status, created_at, sent_at
FROM evnt_evhk_itxeb_out
ORDER BY id DESC

```

Absence de pollution `root` / `crs` :

```sql

       user_id, status, created_at, sent_at
```

## Scénario fonctionnel validé

Exemple validé sur le cours `194` :

```text
course_ref_id 194 / course_obj_id 629 / enabled 1
file ref_id 196 / obj_id 636 / enabled 1
htlm ref_id 207 / obj_id 655 / enabled 1
```

Résultat outbox attendu :

```text
file ref_id 196 -> file_downloaded -> sent
htlm ref_id 207 -> repository_object_access -> sent
```

Test de refus :

```sql
SELECT MAX(id) AS max_outbox_id
FROM evnt_evhk_itxeb_out;
```

Après consultation d'une ressource désactivée :

```sql
SELECT id, event_log_id, event_type, obj_type, ref_id, obj_id,
       user_id, status, created_at, sent_at
FROM evnt_evhk_itxeb_out
WHERE id > <MAX_ID_AVANT_TEST>
ORDER BY id ASC;
```

Résultat attendu :

```text
Empty set
```

## Critères de stabilisation

La V0.7 peut être taguée si :

- la syntaxe PHP est OK ;
- le build ILIAS est OK ;
- l'écran admin plugin permet d'ouvrir le cours par `course_ref_id` ;
- l'écran enregistre `ccfg` et `rcfg` ;
- une ressource activée génère une ligne outbox envoyée ;
- une ressource désactivée ne génère aucune ligne outbox ;
- les enrichissements xAPI V0.6 sont conservés ;
- aucun `root` ou `crs` parasite n'est généré ;
- Windows et serveur sont alignés sur le même commit `origin/v0.7`.

## Préparation du tag v0.7.0

À faire uniquement après validation finale :

