# V0.7 — Configuration xAPI par cours et par ressource

Ce document décrit le socle technique ajouté en V0.7 pour permettre, dans les lots suivants, à un administrateur ou responsable de cours d'activer les traces xAPI au niveau du cours et de choisir les ressources tracées.

## État de ce lot

Ce lot ajoute uniquement :

- les tables SQL de configuration ;
- un repository PHP de lecture/écriture ;
- les requêtes de vérification.

Il ne modifie pas encore :

- l'interface dans l'objet cours ;
- la génération des statements ;
- le filtrage avant outbox.

## Limite ILIAS sur les noms de tables

ILIAS limite les noms de tables à 22 caractères.

Les noms longs initialement envisagés ne sont donc pas utilisés. Les noms retenus sont :

```text
evnt_evhk_itxeb_ccfg  configuration cours
evnt_evhk_itxeb_rcfg  configuration ressources
```

## Tables ajoutées

### `evnt_evhk_itxeb_ccfg`

Configuration au niveau du cours.

Colonnes :

```text
course_ref_id  ref_id du cours ILIAS, clé primaire
course_obj_id  obj_id du cours ILIAS
enabled        1 = xAPI activé pour le cours, 0 = désactivé
created_at     date de création de la configuration
updated_at     date de dernière modification
updated_by     utilisateur ayant modifié la configuration
```

Règle cible V0.7 :

```text
absence de ligne = cours non configuré = tracking xAPI désactivé par défaut
```

### `evnt_evhk_itxeb_rcfg`

Configuration au niveau des ressources d'un cours.

Colonnes :

```text
course_ref_id  ref_id du cours parent
ref_id         ref_id de la ressource
obj_id         obj_id de la ressource
obj_type       type ILIAS de la ressource
enabled        1 = ressource traçable, 0 = ressource désactivée
created_at     date de création de la configuration
updated_at     date de dernière modification
updated_by     utilisateur ayant modifié la configuration
```

Clé primaire :

```text
(course_ref_id, ref_id)
```

## Repository PHP

Classe ajoutée :

```text
classes/class.ilIliasTraxEventBridgeCourseTrackingRepository.php
```

Constantes de tables :

```text
COURSE_TABLE   = evnt_evhk_itxeb_ccfg
RESOURCE_TABLE = evnt_evhk_itxeb_rcfg
```

Méthodes principales :

```text
courseTableExists()
resourceTableExists()
tablesExist()
getCourseConfig($courseRefId)
isCourseConfigured($courseRefId)
isCourseEnabled($courseRefId)
setCourseEnabled($courseRefId, $courseObjId, $enabled, $updatedBy)
findResourceConfigs($courseRefId)
getResourceConfig($courseRefId, $refId)
isResourceConfigured($courseRefId, $refId)
isResourceEnabled($courseRefId, $refId)
setResourceEnabled($courseRefId, $refId, $objId, $objType, $enabled, $updatedBy)
setResourcesEnabled($courseRefId, $resources, $updatedBy)
findEnabledResourceRefIds($courseRefId)
deleteCourseConfig($courseRefId)
```

Ce repository sera utilisé par les lots suivants :

- résolution des ressources du cours ;
- interface cours ;
- filtrage avant insertion outbox.


## Tests SQL manuels temporaires

Insérer une configuration de cours :

```sql
INSERT INTO evnt_evhk_itxeb_ccfg
(course_ref_id, course_obj_id, enabled, created_at, updated_at, updated_by)
VALUES
(177, 594, 1, NOW(), NOW(), 6)
ON DUPLICATE KEY UPDATE
course_obj_id = VALUES(course_obj_id),
enabled = VALUES(enabled),
updated_at = VALUES(updated_at),
updated_by = VALUES(updated_by);
```

Insérer une configuration de ressource :

```sql
INSERT INTO evnt_evhk_itxeb_rcfg
(course_ref_id, ref_id, obj_id, obj_type, enabled, created_at, updated_at, updated_by)
VALUES
(177, 182, 605, 'wiki', 1, NOW(), NOW(), 6)
ON DUPLICATE KEY UPDATE
obj_id = VALUES(obj_id),
obj_type = VALUES(obj_type),
enabled = VALUES(enabled),
updated_at = VALUES(updated_at),
updated_by = VALUES(updated_by);
```

Lire la configuration :

```sql
SELECT * FROM evnt_evhk_itxeb_ccfg ORDER BY course_ref_id;
SELECT * FROM evnt_evhk_itxeb_rcfg ORDER BY course_ref_id, ref_id;
```

Nettoyer le test manuel si nécessaire :

```sql
DELETE FROM evnt_evhk_itxeb_rcfg WHERE course_ref_id = 177;
DELETE FROM evnt_evhk_itxeb_ccfg WHERE course_ref_id = 177;
```

## Points importants

À ce stade, les tables peuvent être alimentées mais ne pilotent pas encore les statements xAPI.

Le comportement V0.6 reste donc inchangé tant que le lot de filtrage n'est pas implémenté.
