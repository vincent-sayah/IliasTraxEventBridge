# V0.12 — Enrichissement pédagogique du tableau de bord

## Statut

| Élément | Valeur |
|---|---|
| Branche | `v0.12-dashboard-pedagogique` |
| Base | `main` / `v0.11.0` |
| Version cible | `0.12.0` |
| Type | Évolution fonctionnelle pédagogique |
| Source des données | TRAX/LRS |
| Rôle de l'outbox locale | File technique d'envoi uniquement |

## Objectif

La V0.12 vise à transformer le suivi xAPI en outil d'aide au pilotage pédagogique.

La V0.11 a sécurisé l'exploitation et le diagnostic. La V0.12 doit maintenant aider un formateur, un concepteur ou un pilote de cours à comprendre rapidement :

- quelles ressources sont réellement utilisées ;
- quelles ressources sont ignorées ;
- où les apprenants échouent ;
- quelles activités progressent ou régressent ;
- quels signaux faibles doivent attirer l'attention ;
- quelles actions pédagogiques peuvent être envisagées.

## Principe conservé

La décision d'architecture reste inchangée :

```text
Outbox locale = file technique d'envoi
TRAX/LRS      = source officielle du suivi xAPI pédagogique
```

Toutes les vues pédagogiques doivent rester alimentées par TRAX/LRS.

## Périmètre fonctionnel V0.12

### 1. Indicateurs pédagogiques synthétiques

Ajouter ou améliorer les indicateurs suivants dans le tableau de bord :

- nombre d'apprenants actifs ;
- taux d'activité du cours ;
- ressources consultées ;
- ressources non consultées ;
- ressources à surveiller ;
- tests ou quiz avec échecs fréquents ;
- score moyen ;
- évolution par rapport à la période précédente.

### 2. Ressources à surveiller

Détecter les ressources présentant un signal pédagogique faible ou négatif :

- ressource activée sans aucune trace ;
- ressource très consultée mais associée à des échecs ;
- ressource avec activité en baisse ;
- test avec taux d'échec élevé ;
- test avec score moyen faible.

### 3. Synthèse lisible pour formateur

Ajouter un bloc de synthèse rédigé simplement, par exemple :

```text
3 ressources semblent peu utilisées.
2 tests présentent un taux d'échec élevé.
L'activité du cours est en hausse sur les 7 derniers jours.
```

Ce bloc doit rester déterministe, sans IA dans cette version.

### 4. Comparaison de périodes

Améliorer la comparaison entre période courante et période précédente :

- activité globale ;
- apprenants actifs ;
- ressources consultées ;
- score moyen ;
- nombre de réussites ;
- nombre d'échecs.

### 5. Vue Analyse enrichie

Dans l'onglet `Analyse`, améliorer la lecture par ressource :

- statut pédagogique de la ressource ;
- nombre de statements ;
- apprenants distincts ;
- score moyen si disponible ;
- réussite / échec si disponible ;
- dernier événement ;
- signal `OK`, `à surveiller`, `critique`.

### 6. Export CSV enrichi

L'export CSV Expert doit conserver les colonnes techniques utiles, mais peut ajouter des colonnes pédagogiques calculées si elles sont disponibles :

- période ;
- type de ressource ;
- statut pédagogique ;
- score ;
- success ;
- completion ;
- statement id ;
- actor anonymisé ou pseudonymisé selon le comportement existant.

## Hors périmètre V0.12

La V0.12 ne doit pas intégrer :

- analyse IA ;
- appel API externe IA ;
- stockage long terme hors TRAX/LRS ;
- modification du modèle de sécurité ;
- changement de protocole xAPI ;
- dépendance fonctionnelle à l'outbox locale.

L'analyse IA est prévue plus tard dans la roadmap, probablement V0.13.

## Points techniques à inspecter

Avant développement, inspecter notamment :

- `classes/class.ilIliasTraxEventBridgeLrsCourseSummary.php` ;
- les classes du plugin compagnon UIHook ;
- les méthodes de rendu du tableau de bord ;
- les méthodes de rendu de l'onglet Analyse ;
- les méthodes d'export CSV ;
- les filtres de période déjà existants.

## Critères d'acceptation

La V0.12 pourra être considérée comme validée si :

- la version plugin passe à `0.12.0` ;
- les vues restent alimentées par TRAX/LRS ;
- le tableau de bord présente des indicateurs plus lisibles ;
- l'onglet Analyse met en évidence les ressources à surveiller ;
- aucune régression n'est observée sur l'accès `Cours > Suivi xAPI` ;
- l'export CSV Expert fonctionne toujours ;
- les diagnostics V0.11 restent disponibles ;
- la documentation est mise à jour.

## Plan de développement proposé

1. Inspecter les classes de synthèse LRS existantes.
2. Identifier les données déjà disponibles sans nouvelle requête lourde.
3. Ajouter une structure de synthèse pédagogique déterministe.
4. Ajouter les statuts `OK`, `à surveiller`, `critique`.
5. Modifier le rendu du tableau de bord.
6. Modifier le rendu de l'onglet Analyse.
7. Mettre à jour la documentation.
8. Valider sur VM ILIAS.
