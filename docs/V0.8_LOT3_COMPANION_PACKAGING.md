# V0.8 lot 3 — packaging propre du plugin compagnon UIHook

## Objectif

Le lot 3 corrige les warnings Composer observés après installation du plugin compagnon `IliasTraxEventBridgeCourseUI` :

```text
Ambiguous class resolution, "ilIliasTraxEventBridgeCourseUIPlugin" was found in both ...
```

Cause : les classes PHP du compagnon étaient présentes à deux endroits :

```text
1. dans le dépôt principal :
   EventHandling/EventHook/IliasTraxEventBridge/companion/IliasTraxEventBridgeCourseUI/...

2. dans le slot actif ILIAS :
   UIComponent/UserInterfaceHook/IliasTraxEventBridgeCourseUI/...
```

Composer scanne les deux emplacements et détecte deux déclarations de classes identiques.

## Solution retenue

Le dépôt principal ne contient plus les fichiers PHP générés du compagnon.

Les sources du compagnon sont conservées sous forme de templates :

```text
companion/IliasTraxEventBridgeCourseUI/plugin.php.tpl
companion/IliasTraxEventBridgeCourseUI/classes/*.php.tpl
```

Ces fichiers `.tpl` ne sont pas interprétés par Composer comme des fichiers PHP de classes.

Un script d'installation matérialise les templates en vrais fichiers `.php` uniquement dans le slot ILIAS actif :

```text
scripts/install_course_ui_companion.sh
```

## Installation du compagnon

```bash
cd /var/www/ilias/public/Customizing/global/plugins/Services/EventHandling/EventHook/IliasTraxEventBridge

bash scripts/install_course_ui_companion.sh
```

Variables optionnelles :

```bash
ILIAS_ROOT=/var/www/ilias
HTTPD_USER=apache
```

Exemple :

```bash
ILIAS_ROOT=/var/www/ilias HTTPD_USER=apache bash scripts/install_course_ui_companion.sh
```

Le script :

1. supprime l'ancien dossier actif du compagnon ;
2. recrée le dossier cible ;
3. copie les `.php.tpl` en `.php` ;
4. applique les droits ;
5. lance `php -l` sur les fichiers générés.

## Chemin cible

```text
/var/www/ilias/public/Customizing/global/plugins/Services/UIComponent/UserInterfaceHook/IliasTraxEventBridgeCourseUI
```

## Validation syntaxe

```bash
cd /var/www/ilias/public/Customizing/global/plugins/Services/EventHandling/EventHook/IliasTraxEventBridge
find . -name "*.php" -print0 | xargs -0 -n1 php -l
```

Cette commande ne doit plus analyser les classes du compagnon dans le dossier `companion/`, car elles sont désormais en `.php.tpl`.

Contrôle du dossier source :

```bash
find companion/IliasTraxEventBridgeCourseUI -name "*.php" -print
```

Résultat attendu : aucune ligne.

Contrôle du dossier cible actif :

```bash
find /var/www/ilias/public/Customizing/global/plugins/Services/UIComponent/UserInterfaceHook/IliasTraxEventBridgeCourseUI -name "*.php" -print
```

Résultat attendu : les fichiers PHP actifs du plugin compagnon.

## Validation Composer

Après installation du compagnon :

```bash
cd /var/www/ilias
sudo -u apache composer du
```

Les warnings suivants ne doivent plus apparaître :

```text
Ambiguous class resolution, "ilIliasTraxEventBridgeCourseUIPlugin" ...
Ambiguous class resolution, "ilIliasTraxEventBridgeCourseUIScreen" ...
Ambiguous class resolution, "ilIliasTraxEventBridgeCourseUIBridge" ...
Ambiguous class resolution, "ilIliasTraxEventBridgeCourseUIUIHookGUI" ...
```

Des warnings ILIAS indépendants peuvent rester, par exemple ceux liés aux exemples `scripts/PHP-CS-Fixer`.

## Validation fonctionnelle

Après rebuild :

```bash
cd /var/www/ilias
sudo -u apache php cli/setup.php build --yes
systemctl restart httpd
```

Contrôles ILIAS :

1. `IliasTraxEventBridgeCourseUI` est toujours présent dans les plugins UIHook ;
2. le plugin compagnon est actif ;
3. dans un cours, le sous-onglet `Paramètres > Suivi xAPI` est toujours présent ;
4. l'écran s'affiche dans le contenu central ;
5. l'enregistrement de la configuration xAPI du cours fonctionne.
