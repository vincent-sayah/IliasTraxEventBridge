# Bilan V0.5 — périmètre cours

## Statut

La V0.5 est stabilisée en **v0.5.5**.

## Objectif réalisé

La V0.5 limite l'envoi xAPI aux objets situés dans un cours ILIAS.

Un objet créé ou consulté directement dans une catégorie, un dossier hors cours ou un autre contexte non cours ne crée pas de statement envoyé vers TRAX.

## Règles métier validées

1. Le journal brut continue à enregistrer les événements dans `evnt_evhk_itxeb_log`.
2. La création d'un statement dans l'outbox est bloquée si aucun cours parent n'est trouvé.
3. Le cours parent est identifié à partir du `ref_id`, de l'arborescence ILIAS ou du contexte de l'événement.
4. Les consultations réelles sont détectées via `read_event`.
5. La table `evnt_evhk_itxeb_read` évite les doublons de consultation.
6. Les événements génériques `Tracking:updateStatus` sur `crs` ou `root` ne génèrent plus de statements xAPI.

## Types d'objets validés

- test ;
- fichier ;
- blog ;
- forum ;
- lien web ;
- mediacast ;
- wiki ;
- module HTML ;
- module web ;
- module SCORM.

## Validation obtenue

- Un objet placé hors cours ne génère pas de statement envoyé vers TRAX.
- Le même type d'objet placé dans un cours peut générer un statement.
- Les consultations blog, lien web, mediacast, wiki et forum sont tracées via `read_event` au passage du cron.
- Les statements sont envoyés vers TRAX via l'outbox et le cron.
- Les traces parasites `root` et `crs` sont absentes de l'outbox après V0.5.5.

## Points reportés

- Activation ou désactivation par cours.
- Choix des types d'objets par cours.
- Enrichissement avancé des statements xAPI en V0.6.
