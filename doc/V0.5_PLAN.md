# Plan V0.5 — périmètre cours

## Objectif

La V0.5 doit limiter l'envoi xAPI aux objets situés dans un cours ILIAS.

Un objet créé directement dans une catégorie, par exemple un test hors cours, ne doit pas créer de statement envoyé vers TRAX.

## Règles métier

1. Le journal brut peut continuer à enregistrer les événements.
2. La création d'un statement dans l'outbox doit être bloquée si aucun cours parent n'est trouvé.
3. Le cours parent doit être identifié à partir du `ref_id`, de l'arborescence ILIAS ou du contexte de l'événement.
4. La configuration du cours doit être appliquée avant la création du statement.

## Paramètres de cours attendus

Ajouter une configuration accessible à l'administrateur du cours :

- activer ou désactiver l'envoi xAPI vers TRAX pour ce cours ;
- choisir les types d'objets activés pour l'envoi xAPI ;
- appliquer ces choix aux objets contenus dans le cours.

## Types d'objets à couvrir

Déjà couverts en V0.4 : test et fichier.

À ajouter en V0.5 : blog, forum, lien web, mediacast, wiki, module web et module SCORM.

## Validation attendue

- Un test placé dans une catégorie ne doit pas être envoyé vers TRAX.
- Le même test placé dans un cours peut être envoyé si le cours l'autorise.
- Un cours avec xAPI désactivé ne génère aucun statement.
- Un cours avec xAPI activé uniquement pour certains types d'objets ne génère que les statements autorisés.
